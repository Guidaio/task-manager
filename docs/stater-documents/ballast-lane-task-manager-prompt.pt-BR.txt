SIGNALR + BACKGROUND QUEUE
Objetivo:
Quando o usuário criar, atualizar ou deletar uma task, a API deve concluir a operação principal e enfileirar uma notificação.
Um BackgroundService processa essa fila e envia a mensagem para o frontend via SignalR.

Fluxo:
1. Usuário executa ação no Angular.
2. API valida request.
3. Application service executa regra.
4. Repository persiste dados.
5. Notification é persistida ou criada.
6. Uma mensagem é enfileirada em Channel<T>.
7. Endpoint retorna resposta.
8. BackgroundService consome a fila.
9. SignalR envia notificação para o usuário conectado.
10. Frontend exibe toast/banner.

Importante:
- SignalR deve ser usado para notificação em tempo real.
- A fila pode ser em memória com Channel<T>.
- Não usar Azure Service Bus real neste teste.
- Documentar que, em produção, a fila em memória poderia ser substituída por Azure Service Bus, RabbitMQ ou outro broker durável.
